An everyday example of control flow would be an enthusiastic web development student coming across a glitchy and soul destroying website that may rhyme with Dodecademy. The student tries to put in the input that the website wants them to, but the website does not accept it. So in this control flow, the "if" statement is if the student gets it right, codecademy will do the output, which is a green button of relief, rarely experienced on this harsh website, and this will break the loop. The far more frequent alternative, "else" is when the student has left out a semicolon, space or nothing at all but for some reason the website still chooses not to accept the code. In this hypothetical control flow loop, else will return us to the start of the loop, where student then has to input a different set of code. This will repeat until either the code is right, or the student feels like this about the website, and skips on to the next exercise.
